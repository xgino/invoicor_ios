## Config: Dev vs Prod

The `.xcconfig` files set the API URL at build time. Xcode uses different configs for Debug (dev) and Release (prod) schemes.

**Dev.xcconfig:**
```
API_BASE_URL = http:$(SLASH)$(SLASH)localhost:8000/api
```

**Prod.xcconfig:**
```
API_BASE_URL = https:$(SLASH)$(SLASH)api.invoicor.com/api
```

**Info.plist** references it: `API_BASE_URL = $(API_BASE_URL)`

**AppConfig.swift** reads it at runtime: reads `API_BASE_URL` from `Bundle.main.infoDictionary`. This way you change the URL by switching the Xcode scheme (Debug/Release), never by editing code.

## Assets.xcassets — What Goes Here

| Asset | What it is | Size / Format |
|-------|-----------|---------------|
| `AppIcon` | App icon shown on home screen + App Store | 1024×1024 PNG, Xcode generates all sizes |
| `AccentColor` | Primary tint color for buttons, links, tab bar | One color, light+dark variants |
| `BrandBlue` | The prominent blue for the center + button | Hex value |
| `CardBG` | Background for stat cards on dashboard | Light gray in light mode, dark gray in dark mode |
| `SubtleGray` | Secondary text, borders, dividers | Adapts to light/dark |
| `logo` | Logo image on welcome screen | PNG or PDF vector, ~200pt wide |
| `empty-state` | Illustration for "no invoices yet" | PNG or SF Symbol reference |

You create these in Xcode by clicking on `Assets.xcassets` in the navigator, then the + button at the bottom. Drag images in. Colors are created as "New Color Set."

## Build Order

| Step | What to build | Files touched | Simulator check |
|------|--------------|---------------|-----------------|
| **1** | Core layer | `AppConfig.swift`, `Models.swift`, `APIClient.swift`, `AuthManager.swift` | App launches, no crash |
| **2** | Login flow | `LoginPage.swift`, `RootView.swift` | Register → login → see tab bar |
| **3** | Tab bar + stubs | `RootView.swift` (add MainTabView), all Page files as `Text()` placeholders | Tap between 5 tabs |
| **4** | Dashboard | `HomePage.swift`, `StatusBadge.swift` | See stats + recent invoices from API |
| **5** | Invoice list + preview | `InvoicesPage.swift`, `SVGView.swift` | See list, tap one, see rendered SVG |
| **6** | Create invoice | `CreatePage.swift` | Create invoice, see it in list |
| **7** | Clients | `AssetsPage.swift` | Add/edit/delete clients |
| **8** | Settings | `SettingsPage.swift` | Edit profile, logout, delete account |
| **9** | Onboarding | `SetupPage.swift` | Fresh install: business setup → template pick → dashboard |
| **10** | Polish | All files | Empty states, error handling, loading spinners |

**Bottom bar changes by status:**

| Status | Buttons shown |
|--------|--------------|
| Draft | Edit, Share, Mark Sent |
| Sent | Share, Mark Paid, More (Duplicate) |
| Paid | Share, More (Duplicate) |
| Overdue | Share, Mark Paid, More (Duplicate) |
| Cancelled | Share, More (Duplicate) |

"More" is a `...` menu button that shows Duplicate (and Delete for drafts). This keeps the bar clean — max 3 visible buttons.

**Share** converts SVG to PDF using iOS native rendering, opens `UIActivityViewController`.

**After sharing a draft:** auto-marks as "sent", re-fetches SVG, updates badge.

## Data Flow

```
API  ←→  APIClient  ←→  ViewModel  ←→  View

- View calls ViewModel methods (user taps button)
- ViewModel calls APIClient (async/await)
- APIClient calls the API (URLSession)
- Response decoded into Model structs
- ViewModel updates @Published properties
- View re-renders automatically (SwiftUI reactivity)
```

ViewModels use `@Observable` (iOS 17+) so views automatically update when ViewModel properties change. No manual `objectWillChange.send()` needed.

For small pages (Settings, Assets), the ViewModel logic can live directly in the View using `@State`. Only use a separate ViewModel class when the logic is complex (Dashboard, Create Invoice, Invoice List).
